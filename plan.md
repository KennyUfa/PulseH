Хорошо, давайте декомпозируем проект PulseHR на небольшие этапы, ориентируясь на использование Django REST Framework (DRF) для backend и Vue.js для frontend.

**Этап 1: Подготовка и проектирование**

*   **1.1. Создание репозиториев:** Настройте репозитории для backend (Django) и frontend (Vue) на GitHub/GitLab.
*   **1.2. Определение моделей данных (Django):**
    *   `User` (можно использовать стандартную модель Django, расширенную через `AbstractUser` или `OneToOneField`, если нужно хранить доп. информацию типа роли, телефона, часового пояса).
    *   `Survey` (название, описание, даты начала/окончания, статус, режим анонимности, аудитория).
    *   `Question` (текст вопроса, тип, порядок в опросе, ссылка на опрос).
    *   `AnswerOption` (для выбора - текст, порядок, ссылка на вопрос).
    *   `Response` (ссылка на пользователя, ссылка на опрос, дата/время).
    *   `Answer` (ссылка на `Response`, ссылка на `Question`, фактический ответ, ссылка на `AnswerOption` если применимо).
    *   `SurveyTarget` (для определения целевой аудитории по ролям/подразделениям).
    *   `PushSubscription` (для Web Push - user, endpoint, p256dh, auth).
    *   `NotificationLog` (для журналирования отправок - тип, содержание, получатель, статус, время).
*   **1.3. Проектирование API (Django REST):** Определите начальный набор эндпоинтов (URLs) и сериализаторов для основных моделей.
*   **1.4. Создание базового проекта Django:** Установите Django, DRF, настройте базовую конфигурацию (база данных, настройки безопасности и т.д.).

**Этап 2: Базовая аутентификация и авторизация**

*   **2.1. Настройка аутентификации по телефону:** Реализуйте систему регистрации/входа с использованием номера телефона и OTP-кода (SMS или имитация). Это может потребовать интеграции с SMS-провайдером или использования mock-сервиса для MVP.
*   **2.2. Управление сессией:** Обеспечьте сохранение сессии после успешной аутентификации (например, через JWT токены или стандартные сессии Django).
*   **2.3. Роли пользователей:** Реализуйте простую систему ролей (например, `is_hr`, `is_employee`) и соответствующие проверки разрешений в DRF ViewSets.

**Этап 3: Фронтенд (Vue.js) - Базовый интерфейс**

*   **3.1. Создание проекта Vue:** Инициализируйте проект Vue.js (желательно с Vue CLI).
*   **3.2. Настройка маршрутизации:** Используйте `vue-router` для создания основных страниц (Логин/Регистрация, Личный кабинет сотрудника, Панель HR).
*   **3.3. Интеграция с API:** Настройте взаимодействие с backend API (например, с помощью `axios`). Реализуйте формы для аутентификации.
*   **3.4. Базовая верстка:** Создайте основные компоненты и страницы для отображения информации.

**Этап 4: Конструктор опросов (часть HR-панели)**

*   **4.1. Модели Django:** Завершите создание моделей `Survey`, `Question`, `AnswerOption`.
*   **4.2. API для опросов:** Создайте ViewSets и сериализаторы для CRUD операций над опросами, вопросами и вариантами ответов.
*   **4.3. Фронтенд HR:** Разработайте интерфейс для HR-специалиста для создания и редактирования опросов. Позвольте добавлять вопросы разных типов, задавать параметры опроса (название, даты, анонимность, аудитория).

**Этап 5: Прохождение опроса (часть сотрудника)**

*   **5.1. Модели Django:** Завершите модели `Response`, `Answer`. Добавьте логику для проверки, что пользователь не проходил уже данный опрос.
*   **5.2. API для прохождения:** Создайте эндпоинты для получения списка доступных опросов для текущего пользователя и отправки ответов.
*   **5.3. Фронтенд сотрудника:** Разработайте интерфейс для просмотра доступных опросов и прохождения их. Отобразите вопросы в соответствии с их типом и настройками (например, одиночный/множественный выбор, шкала, текстовое поле).

**Этап 6: Ветвление сценариев (Conditional Logic)**

*   **6.1. Расширение моделей Django:** Добавьте поля к моделям `Question` и `AnswerOption`, чтобы определять условия отображения (например, `show_if_question`, `show_if_answer_option`).
*   **6.2. Логика отображения (Frontend/Backend):** Реализуйте логику, которая будет определять, какие вопросы показывать следующими на основе предыдущих ответов. Это может быть частично на фронтенде (после получения всех вопросов опроса) или с участием backend (отправка промежуточных ответов и получение следующего вопроса).
*   **6.3. Тестирование ветвления:** Убедитесь, что логика работает корректно в различных сценариях.

**Этап 7: Режимы конфиденциальности**

*   **7.1. Логика Django:** Убедитесь, что при сохранении ответов (`Response`, `Answer`) в анонимном режиме информация о пользователе не привязывается к этим объектам.
*   **7.2. Фронтенд:** Отобразите соответствующее уведомление пользователю перед началом анонимного опроса.

**Этап 8: Уведомления - Web Push (Часть 1)**

*   **8.1. Настройка Service Worker (Frontend):** Создайте файл `sw.js`, зарегистрируйте его в основном приложении Vue. Реализуйте логику подписки на push-уведомления (`navigator.serviceWorker.register`, `PushManager.subscribe`) и обработку события `push`.
*   **8.2. Управление подпиской (Frontend):** Добавьте UI для запроса разрешения на уведомления и отправки данных подписки (endpoint, keys) на backend.
*   **8.3. Модель подписки (Django):** Создайте модель `PushSubscription` и соответствующий API эндпоинт для сохранения данных подписки.
*   **8.4. Генерация VAPID-ключей (Django):** Сгенерируйте и безопасно храните VAPID-ключи (публичный/приватный) для использования в протоколе Web Push.

**Этап 9: Уведомления - Отправка Web Push (Часть 2)**

*   **9.1. Отправка уведомлений (Django):** Используйте библиотеку `pywebpush` для отправки push-уведомлений по сохранённым endpoint и ключам подписчиков.
*   **9.2. Триггеры отправки (Django):** Реализуйте логику (например, сигналы Django или задачи Celery), которая будет запускать отправку уведомлений при публикации новых опросов или других триггерах.
*   **9.3. Асинхронность (Django):** Настройте очередь задач (Celery + Redis) для асинхронной отправки большого количества уведомлений, чтобы не блокировать основной поток выполнения.
*   **9.4. Очистка подписок (Django):** Добавьте задачу или логику для проверки и удаления недействительных endpoint-ов.

**Этап 10: Уведомления - SMS (Резервный канал)**

*   **10.1. Интеграция с SMS-провайдером (Django):** Выберите провайдера (например, SMSC.ru) и реализуйте отправку SMS через их HTTP API.
*   **10.2. Каскадная логика (Django):** Реализуйте логику проверки статуса Web Push подписки сотрудника. Если подписка отсутствует или устарела, отправляйте SMS.
*   **10.3. Антиспам и ограничения (Django):** Реализуйте проверки на количество отправленных уведомлений (на пользователя, в сутки), отправку только в рабочие часы.
*   **10.4. Журналирование (Django):** Сохраняйте информацию об отправках SMS в `NotificationLog`.

**Этап 11: Аналитика и отчетность (часть HR-панели)**

*   **11.1. Логика Django:** Напишите методы/функции для вычисления метрик (процент прохождения, eNPS, распределение ответов).
*   **11.2. API для аналитики:** Создайте специальные эндпоинты для получения аналитических данных (например, `/api/surveys/{id}/analytics/`, `/api/surveys/{id}/responses/`).
*   **11.3. Фронтенд HR:** Разработайте дашборд для отображения графиков, таблиц и метрик. Используйте библиотеку визуализации данных (например, Chart.js, D3.js).
*   **11.4. Экспорт (Django):** Реализуйте функции экспорта результатов в Excel/CSV (через `django-excel` или `openpyxl`).

**Этап 12: Улучшения и дополнения (MVP+)**

*   **12.1. Управление уведомлениями (Frontend/Django):** Добавьте UI в личном кабинете сотрудника для управления предпочтениями уведомлений (вкл/выкл каналов, "не беспокоить").
*   **12.2. Дополнительные каналы (Telegram Bot API):** (Если есть время) Начните интеграцию с Telegram Bot API для отправки уведомлений.
*   **12.3. PWA (Frontend):** Добавьте `manifest.json` и улучшите `sw.js` для поддержки PWA функций (установка, работа в offline для кэшированных ресурсов).
*   **12.4. Тестирование:** Напишите модульные и интеграционные тесты для основных компонентов.
*   **12.5. Документация (README.md):** Подготовьте подробный README файл с описанием проекта, инструкцией по запуску, списком технологий и команды.

Этот план предоставляет пошаговый путь от идеи до MVP PulseHR, разбитый на управляемые этапы. Начните с первых этапов и постепенно добавляйте функциональность.
